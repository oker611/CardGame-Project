# 锄大地 AI 出牌策略设计文档

## 1. 引言

锄大地是一种四人参与的纸牌游戏，牌型丰富，且玩家只能看到自己的手牌，属于不完全信息博弈。本项目的目标是为游戏中的 AI 玩家开发一套出牌策略，这些 AI 在每局比赛中可以部署 1 到 3 个，与人类或其他 AI 同场竞技。文档将按照策略的实现顺序，逐步说明我们采用的架构、每一环节的设计理由、核心算法伪代码，以及最终可预期的成果。

## 2. 设计思路与总体架构

我们的设计遵循“离线训练、在线轻量决策”的原则。在线对战时，AI 需要在极短时间内做出反应，因此我们将复杂的参数搜索和知识提炼放在离线阶段完成；而在线决策则依靠蒙特卡洛模拟来推演牌局走向，并用启发式规则和有限状态机来注入牌理经验。整体策略的实现可以分解为五个递进的步骤：第一步，从手牌中生成候选出牌动作；第二步，为对手构建可能的手牌分布；第三步，通过蒙特卡洛模拟评估每个候选动作的长期收益；第四步，引入阶段管理来修正评估结果；第五步，利用遗传算法离线优化上述所有环节中的自由参数。此外，我们还引入大语言模型作为离线辅助，用来从牌谱中提炼战术知识。以下将按此顺序逐一展开。

## 3. 策略实现步骤

### 3.1 第一步：手牌拆解与候选动作生成

每一轮轮到 AI 出牌时，它首先要做的是看清自己手中的牌，并找出所有可以打出的合法牌型。这一步是整个决策链条的起点，必须既全面又高效。

我们采用基于牌型枚举的启发式方法。AI 会依照锄大地的牌型规则，依次尝试从手牌中提取单张、对子、三张、顺子、同花顺等所有可能的组合。随后，根据上一轮打出的牌（类型和大小）过滤出当前可以合法压过的牌型。如果没有任何可出的牌，则“过牌”会自动成为唯一的候选动作。

为了避免蒙特卡洛模拟阶段的计算量过大，这里还会进行一次快速的启发式初筛。我们会给每一个候选动作打一个基础分，该分数综合考虑了出手的牌力强弱、留下的手牌形状是否健康、以及是否保留了关键的控制牌（如单张 2 或大顺子）。然后只保留分数最高的前若干名候选（通常 10 到 15 个）进入后续的精确模拟。这样做大幅压缩了搜索空间，却几乎不会损失决策质量，因为大量明显劣质的动作已在早期被剔除。

**伪代码**：

```
function generate_candidates(hand, last_play):
    all_combinations = []
    for play_type in [SINGLE, PAIR, TRIPLE, STRAIGHT, ...]:
        all_combinations += extract_plays(hand, play_type)
    legal_plays = filter by last_play's type and rank
    if legal_plays is empty:
        legal_plays = [PASS]
    sort legal_plays by heuristic_score(hand, play)
    return top_k(legal_plays, K=15)
```

### 3.2 第二步：对手手牌范围的建模

锄大地的核心挑战在于，AI 永远无法确知其他三人手中的牌。为了做出有远见的决策，AI 必须“猜测”剩余牌的分布。我们采用均匀随机采样的方法来构造多个可能的世界状态。

具体做法是：从一副完整的牌中去掉 AI 自己的手牌以及已经打出过的牌，剩下的牌称为“剩余牌池”。然后，根据已知的各家剩余手牌张数（可从历史出牌记录中累加得出），将剩余牌池中的牌打乱顺序后依次分配给三个对手，形成三份手牌。重复这一过程上百次，就得到了一组对手手牌的采样集合。这种均匀随机采样虽然简单，但在缺乏强假设的情况下是最无偏的估计，能够很好地支撑蒙特卡洛模拟。

**伪代码**：

```
function sample_worlds(our_hand, history, num_samples):
    remaining = all cards - our_hand - cards seen in history
    worlds = []
    for i=1 to num_samples:
        shuffled = random_shuffle(remaining)
        opp1 = shuffled[0 : N1]
        opp2 = shuffled[N1 : N1+N2]
        opp3 = shuffled[N1+N2 : end]
        worlds.append( (opp1, opp2, opp3) )
    return worlds
```

### 3.3 第三步：蒙特卡洛模拟推演

有了候选动作和可能的对手手牌世界，接下来就需要回答一个关键问题：如果我现在打出这张牌，整局游戏最终会对我有多有利？蒙特卡洛模拟正是用来回答这个问题的工具。

对每一个候选动作，AI 会在每一个采样到的对手手牌世界中，将游戏从当前状态一直模拟到结束。模拟中所有玩家（包括对手和模拟中的自己）都使用一个简化的、速度极快的贪心策略来出牌，确保模拟可以在极短时间内完成。当一局模拟结束时，我们可以得到 AI 的最终排名（第一至第四名），并将其转换为得分。把一个候选动作在所有采样世界中的平均得分计算出来，就得到了该动作的期望收益。

由于蒙特卡洛模拟直接度量了“动作 → 未来结果”的因果关系，它天然适合不完全信息博弈，而且实现原理清晰，便于调试和优化。

**伪代码**：

```
function evaluate_candidates(hand, candidates, history, num_samples=100):
    worlds = sample_worlds(hand, history, num_samples)
    scores = []
    for each candidate in candidates:
        total_score = 0
        for world in worlds:
            state = setup_game(hand, world, history, candidate)
            simulate_until_end(state, greedy_policy)
            rank = our_final_rank(state)
            total_score += (4 - rank)   // 第一得3分，第四得0分
        avg_score = total_score / num_samples
        scores.append(avg_score)
    return scores
```

### 3.4 第四步：有限状态机（FSM）阶段修正

如果 AI 在对局的所有阶段都使用同一个评估标准，它就会表现得缺乏“大局观”。经验丰富的玩家知道，开局应当保留牌型的灵活性和大牌控制权，中盘要适时抢夺出牌权并压制下家，而到了残局则必须以最快速度脱手为唯一目标。因此，我们引入了一个简单的有限状态机来管理这三种阶段。

阶段划分的依据是当前手牌数以及大牌的消耗情况。当手牌数较多（如≥13）且关键控制牌尚未露面时，处于开局；当手牌数处于中等范围（如 8~12 张）或部分大牌已打出时，进入中盘；当手牌数减少到 7 张及以下时，则判定为残局。

状态机并不直接决定出什么牌，而是在蒙特卡洛模拟得出的基础分之上，施加一个阶段性的调整分。例如，开局时保留长顺子或大单张可以获得正向加成；残局时如果手牌中还积压着难以打出的孤张大牌，则会受到额外扣分，督促 AI 及早处理。这样一来，同一个候选动作在不同阶段会受到不同的评价，从而使 AI 的整体行为自然地带上了“牌局节奏感”。

**伪代码**：

```
function adjusted_score(candidate, base_score, hand, phase):
    bonus = 0
    if phase == EARLY:
        bonus += 0.2 * count_large_controls(hand)
    elif phase == LATE:
        bonus -= 0.5 * count_unplayed_high_singles(hand)
    else:  // MID
        if candidate can suppress next_player:
            bonus += 0.3
    return base_score + bonus
```

### 3.5 第五步：遗传算法离线参数优化

前面四个步骤构建出了一个完整可用的 AI，但它的性能很大程度取决于内部的一系列自由参数，比如：启发式初筛的权重系数、蒙特卡洛模拟的次数、FSM 切换的手牌阈值、以及各项调整分的大小等。手工调整这些参数不仅费时费力，而且很难在多参数相互作用的空间中找到较优解。为此，我们引入遗传算法来自动完成这项任务。

我们将所有待优化的参数编码成一个一维浮点数数组，称为染色体。一条染色体就代表一个完整的 AI 策略个体。优化开始时，先随机生成一个包含 50 个个体的种群。然后，对每一个个体，让它与一个固定的基准贪心 AI 进行 200 局对战，根据平均每局得分计算适应度。为了保证不同个体之间的可比性，所有个体都会在完全相同的牌谱序列下进行测试。

得到适应度后，种群便开始一代代地进化。每一代我们保留表现最好的两个个体（精英），其余个体则通过锦标赛选择的方式从上一代中挑选出来，并两两进行单点交叉以产生子代。子代还会以一定概率发生基因突变，即对某个参数加上一个小的随机扰动，从而保持种群的多样性。这个过程重复约 50 代，最终适应度最高的个体所携带的参数，就作为最优参数部署到 AI 中。

遗传算法的全部运算都在离线阶段完成，通常利用多线程并行评估，可以将数十小时的训练压缩到一个晚上。一旦训练结束，AI 只需读入这一组最优参数，便可在对战中展现出色的实力。

**伪代码**（进化主循环）：

```
population = random_population(size=50)
for gen = 1 to 50:
    fitness_scores = parallel_evaluate(population)   // 每个个体对战200局
    elites = best 2 individuals
    next_pop = elites
    while size(next_pop) < 50:
        p1 = tournament_select(population, fitness_scores)
        p2 = tournament_select(population, fitness_scores)
        c1, c2 = single_point_crossover(p1, p2)
        mutate(c1); mutate(c2)
        next_pop.append(c1, c2)
    population = next_pop
best_params = population[argmax(fitness_scores)]
save_params(best_params)
```

### 3.6 离线辅助：大语言模型的知识注入

除了通过遗传算法自动发现参数，我们还希望 AI 能够吸收人类玩家的经验与智慧。为此，我们调用 vivo 蓝心大模型等大语言模型，在离线状态下完成三项辅助工作。

第一，战术规则提炼。我们收集若干经典残局牌谱，将手牌和局面描述输入大模型，让它给出最佳出牌建议并解释理由。经过人工校验后，这些解释可以被抽象为新的 FSM 修正条件或启发式初筛规则，从而让 AI 的策略库得到扩充。

第二，对手风格分析。在一局比赛结束后，将某个对手的全部出牌序列发送给大模型，模型会返回对该玩家风格的评语，例如“激进，偏爱短牌组合”或“保守，倾向保留大牌后发制人”。这些标签被存储下来，在下一次对战中可以用来调整蒙特卡洛模拟时该对手的模拟策略偏向，使推演更贴近真实。

第三，开发调试支持。当 AI 的行为出现疑点时，我们可以让大模型审视牌局记录，询问它某一手牌选择是否合理。模型的解释常常能启发我们发现权重设置或逻辑判断中的漏洞，从而加速迭代改进。

所有大模型的调用都是异步、低频的，完全不介入实时出牌循环，因此不会给对局带来任何延迟，但又能有效提升策略的深度和项目的创新性。

## 4. 预期成果

通过上述步骤的实现，我们预期将交付以下成果。首先是完整的软件系统，包含一个可对战的控制台或简易图形界面锄大地程序，以及三种内置 AI：基础贪心 AI、加入 FSM 的增强 AI、搭载蒙特卡洛模拟的全功能 AI。其次是遗传算法训练脚本，能够输出进化曲线图，直观展示从随机参数到强策略的适应度上升过程。如有条件，还会展示大语言模型辅助生成的战术分析样例。

在性能方面，我们预估蒙特卡洛 AI 相较于基础贪心 AI 的胜率能提升 15% 到 25%。经过遗传算法优化参数后的版本，在自对弈中的平均排名将稳定地优于未优化版本。同时，得益于候选动作的剪枝和模拟次数的合理控制，单步决策时间可以控制在 1 秒以内，完全满足实时对战的要求。

最终答辩时，我们能够通过剥离实验来清晰地展示各个模块的贡献：逐一去除 FSM 修正或降低模拟次数，观察胜率的变化，从而证明每一部分的设计价值。进化曲线和大模型协作案例也将作为技术亮点呈现。

## 5. 总结

本文档所描述的锄大地 AI 策略，完全遵循从手牌拆解到决策输出的自然实现流程。我们首先用启发式方法生成候选动作，再通过随机采样构建对手手牌的不确定模型，利用蒙特卡洛模拟推演动作的长期效果，并借助有限状态机赋予 AI 明确的阶段意识。在这些模块之上，遗传算法以数据驱动的方式完成了繁琐的参数寻优工作，大语言模型则为策略库注入了可解释的人类经验。整套方案在线运行轻量、离线训练充实，难度适中且效果可期，能够在实训周期内完整实现并交付一个兼具实力与展示价值的锄大地 AI。
