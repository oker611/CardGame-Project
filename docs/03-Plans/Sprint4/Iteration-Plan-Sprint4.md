## 🧩 卡片8：蓝牙对战最小闭环

### 傅钧烨（负责人）可独立完成

| 子任务 | 说明 |
|--------|------|
| 1. 玩家类型抽象 | 引入 `RemotePlayer`，`GameEngine` 不直接依赖具体玩家类型 |
| 2. 蓝牙连接底层 | 扫描、建立连接、获取输入输出流 |
| 3. 角色分配逻辑 | 房主 / 加入者的本地状态区分 |
| 4. 数据传输协议定义 | JSON 格式定义（开局同步 / 动作同步 / 结束） |
| 5. 发送端实现 | 将本地操作封装为 JSON 并发送 |
| 6. 接收端解析 | 接收 JSON 并解析为可识别的操作 |
| 7. 断线检测与 Toast 提示 | 捕获异常并提示用户 |
| 8.日志输出 | 每次收发内容打印到控制台 |

### 需要傅钧烨 + 陈政昊联调

| 子任务 | 说明 | 双方职责 |
|--------|------|----------|
| 1. 开局手牌同步 | 房主发牌后发送双方手牌，加入者重建状态 | 傅：发送手牌数据<br>陈：提供 `GameEngine` 重建状态的接口 |
| 2. 动作同步驱动本地引擎 | 收到对方出牌/Pass后，本地引擎执行 | 傅：接收并解析动作<br>陈：暴露 `executePlay()` / `executePass()` 接口 |
| 3. 状态一致性校验 | 复用 `PlayValidator` 校验对方动作是否合法 | 傅：调用校验<br>陈：确保 `PlayValidator` 接口稳定 |
| 4. 游戏结束同步 | 一方结束时通知对方 | 傅：发送结束消息<br>陈：提供 `onGameOver()` 回调 |
